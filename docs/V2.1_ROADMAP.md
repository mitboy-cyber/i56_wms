# I56 Framework v2.1 LTS Roadmap

> **Status**: Planning | **Target**: Q3-Q4 2026 | **Based on**: I56 Framework 2.0 (In-Memory Architecture)

---

## Executive Summary

v2.0 delivered a fully functional in-memory WMS platform with 5-module admin panel (OMS/WMS/TMS/CRM/SYS), PDA mobile workflows, AI Runtime integration, and BDL 1.0 design system. v2.1 shifts focus to **production hardening** — replacing in-memory stores with PostgreSQL, enabling multi-tenant isolation, adding event-driven architecture, and building out the plugin/extension system.

## 1. Real Database Layer (PostgreSQL) — P0

### Current State
All 14 business modules use `Mem*Repo` patterns with `sync.RWMutex` and atomic ID counters. Data is lost on restart.

### Target State
- **PostgreSQL 16** as the primary store
- Each `Mem*Repo` gets a parallel `Pg*Repo` implementing the same interface
- Migration system using [golang-migrate](https://github.com/golang-migrate/migrate)
- Connection pooling via `pgxpool`

### Implementation Plan

#### 1.1 SQL Schema Design
```sql
-- Core tables (one per module)
CREATE TABLE tenants (
    id BIGSERIAL PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    slug VARCHAR(50) UNIQUE NOT NULL,
    config JSONB DEFAULT '{}',
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE parcels (
    id BIGSERIAL PRIMARY KEY,
    tenant_id BIGINT NOT NULL REFERENCES tenants(id),
    warehouse_id BIGINT NOT NULL,
    client_id BIGINT NOT NULL,
    tracking_number VARCHAR(100) NOT NULL,
    courier_code VARCHAR(20),
    cargo_type VARCHAR(50) DEFAULT 'general',
    product_name VARCHAR(500),
    actual_weight NUMERIC(10,2),
    length NUMERIC(10,2),
    width NUMERIC(10,2),
    height NUMERIC(10,2),
    status VARCHAR(30) DEFAULT 'pre_declared',
    location_code VARCHAR(100),
    order_id BIGINT,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(tenant_id, tracking_number)
);

-- ... similar for orders, clients, members, warehouses, routes, ledgers, etc.
```

#### 1.2 Repository Interface Compliance
```go
// All repos implement both:
type ParcelRepository interface {
    Create(ctx context.Context, p *domain.Parcel) error
    GetByID(ctx context.Context, tenantID, id int64) (*domain.Parcel, error)
    List(ctx context.Context, tenantID int64, offset, limit int) ([]domain.Parcel, int64, error)
    Update(ctx context.Context, p *domain.Parcel) error
}

// Factory pattern for DI:
func NewParcelRepo(cfg config.Database) ParcelRepository {
    if cfg.Driver == "postgres" {
        return NewPgParcelRepo(cfg)
    }
    return NewMemParcelRepo() // fallback for dev
}
```

#### 1.3 Migration System
```
i56-apps/i56-wms/
└── migrations/
    ├── 000001_create_tenants.up.sql
    ├── 000001_create_tenants.down.sql
    ├── 000002_create_warehouses.up.sql
    ├── 000003_create_parcels.up.sql
    ├── 000004_create_orders.up.sql
    └── ... (one migration per module)
```

#### 1.4 Configuration
```yaml

  driver: postgres
  host: localhost
  port: 5432
  database: i56_v21
  user: i56_app
  password: ${DB_PASSWORD}

    max_conns: 25
```

### Files to Create/Modify

- `i56-modules/*/repository/postgres.go` — Pg*Repo for each module

### Implementation Plan

#### 2.1 Tenant Resolver Middleware
```go
// core/middleware/tenant.go
func TenantResolver(pool *pgxpool.Pool) func(http.Handler) http.Handler {
    return func(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
            // Extract from JWT claims, subdomain, or header
            tenantID := extractTenant(r)
            ctx := context.WithValue(r.Context(), TenantKey, tenantID)
            next.ServeHTTP(w, r.WithContext(ctx))
        })
    }
}
```

## 3. Event Bus Integration — P1

### Current State
No event system. State changes are synchronous and isolated.

### Target State

```
OrderCreated → [Inventory] ReserveStock → [Notification] SendEmail
              → [Workflow]  StartApproval → [Webhook]  NotifyPartner
```

### Implementation Plan


#### 3.2 Event Publisher/Subscriber
```go
// core/events/bus.go

    subscribers map[string][]Handler
}

func (b *Bus) Publish(ctx context.Context, event Event) { ... }
func (b *Bus) Subscribe(eventName string, handler Handler) { ... }
```

#### 3.3 Integration Points
- **OrderService.Create()** → publishes `OrderCreated`
- **ParcelService.Receive()** → publishes `ParcelReceived`
- **ClientRecharge** → publishes `PaymentReceived`

### Files to Create

#### 5.2 DataScope

```go
type DataScope int
const (
    ScopeAll       DataScope = 1  // All data
    ScopeWarehouse DataScope = 2  // Only assigned warehouse
```

#### 5.3 Middleware Enforcement

## Migration Path (v2.0 → v2.1)

### Phase 1: Database Foundation (Weeks 1-4)
1. Set up PostgreSQL with migrations framework
2. Implement `PgParcelRepo` as the first real store
3. Add `PgOrderRepo`, `PgClientRepo`, `PgWarehouseRepo`
4. Add feature flag: `DB_DRIVER=postgres` to switch between mem/pg
5. Data migration tool: `i56-tools migrate --from=memory --to=postgres`

### Phase 2: Multi-Tenant + Security (Weeks 5-7)
1. Tenant model + provisioning API
2. Database-per-tenant router
3. JWT tenant claim injection
4. DataScope enforcement in all repo queries
5. Permission-check middleware on all API routes

### Phase 3: Events + Workflow (Weeks 8-10)
1. Event bus implementation
2. Domain events for Order/Parcel/Payment
3. Workflow engine with approval gates
4. Notification channel adapters (Email/SMS/Line)

### Phase 4: API + Extensions (Weeks 11-13)
1. OpenAPI annotation + Swagger UI
2. Plugin interface + registry
3. Full test suite for PostgreSQL repos
4. Performance testing (100K parcels, 10K orders)
