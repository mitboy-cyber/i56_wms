# I56 Framework — 领域词汇表 (Ubiquitous Language)

> 本文档定义 I56 Framework 中的核心领域术语。所有代码、文档、API 必须使用一致的术语。

---

## 核心概念

| 术语 | 英文 | 定义 |
|------|------|------|
| 租户 | Tenant | 平台上的一个独立企业/组织，数据完全隔离 |
| 仓库 | Warehouse | 物理仓储设施，如"厦门仓" |
| 客户 | Client | 使用平台的终端用户（集运客户），非指 API 客户端 |
| 会员 | Member | 客户的子账户，如收件人/寄件人 |
| 申报人 | Declarant | 负责海关申报的人员，需台湾实名认证 |
| 承运商 | Carrier / Provider | 运输服务提供商（新竹物流/顺丰速运） |
| 快递公司 | Courier | 国内快递（韵达/中通/极兔） |

## 包裹 (Parcel)

| 术语 | 定义 |
|------|------|
| 包裹 | 从大陆卖家寄到仓库的单个快递包裹 |
| 快递单号 | 包裹的快递追踪号 |
| 入库 | 包裹到达仓库 → 签收 → 称重 → 上架 |
| 预申报 | 包裹在入库前已提交海关预审 |
| 已签收 | 仓库已签收包裹 |
| 已称重 | 包裹完成称重，记录实际重量 |
| 已上架 | 包裹放置到指定仓位 |

## 订单 (Order)

| 术语 | 定义 |
|------|------|
| 集运订单 | 客户将多个包裹合并成一个订单发往台湾 |
| 待拣货 | 订单已创建，等待仓库拣货 |
| 待打包 | 包裹已拣齐，等待打包 |
| 待装车 | 包裹已打包，等待装柜 |
| 已装柜 | 包裹已装入集装柜 |
| 清关中 | 正在海关处理 |
| 运输中 | 柜已发运，在途 |
| 已签收 | 台湾端已签收 |
| 已完成 | 订单完结 |

## 物流 (Logistics)

| 术语 | 定义 |
|------|------|
| 线路 | Route，运输路线（厦门→台北） |
| 运输方式 | 空运 / 海快 / 海运 |
| 集装柜 | Container，用于装运的集装箱 |
| 柜号 | 集装柜的编号 |
| 体积重 | 长×宽×高 / 体积系数 |
| 计费重 | max(实际重, 体积重) |

## 财务 (Finance)

| 术语 | 定义 |
|------|------|
| 余额 | 客户账户可用余额 |
| 充值 | 客户向账户充值 |
| 扣费 | 系统自动从账户扣款 |
| 账本 | 所有交易的流水记录 |
| 月结对账单 | 按月度生成的账单汇总 |
| 盈利报表 | 订单/服务/客户/路线维度的盈利分析 |

## 附加服务 (Value-Added Service)

| 术语 | 定义 |
|------|------|
| 加固包装 | 额外加固保护 |
| 验货拍照 | 开箱拍照确认 |
| 合并打包 | 多包裹合并 |
| 退货处理 | 退件服务 |

## 系统 (System)

| 术语 | 定义 |
|------|------|
| 角色 | 权限的集合（超管/仓管/客服/财务/操作员） |
| 权限 | 对某个资源的操作许可 |
| 数据权限 | 限制用户可看到的数据范围 |
| 审计日志 | 所有关键操作的记录 |
| 通知渠道 | 邮件/短信/企业微信/Webhook |
